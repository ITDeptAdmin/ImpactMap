📋 How to Update the Map

1 Download the current Master CSV using the button below.
2 Edit the CSV in Excel — add clinics, update addresses, or fix errors.
3 Upload the updated CSV. GitHub will rebuild the map automatically.
4 Check the Review Queue. If it shows zero, you are done.
5 If rows appear in the Review Queue, check the map location and review the Suggested Fix. Approve it if the location is correct. Reject it and fix the CSV if the location is wrong.
Full Instructions & Help ▸

What this dashboard is

This page is a read-only update center for the Impact Map. It shows the current map data status, the last build time, and any rows that need review.

You cannot directly edit map data from this dashboard. Data changes happen through GitHub so only users with repository write access can update the map.

Uploading the CSV

Use the Upload Updated CSV button below. It opens GitHub's file upload page.

You must be logged in with a GitHub account that has access to this repository.

After you upload the CSV, GitHub automatically rebuilds the map. This usually takes a few minutes. When the build finishes, refresh this dashboard.

Important: Do not rename the CSV file. It must stay named Master_Clinic_ImpactMap.csv.

Green / All Clear status

If the status is green and the Review Queue shows zero rows, no action is needed. The map data is ready.

Review Queue — what it means

Rows appear in the Review Queue when the system could not safely place them on the map without a human check.

A review row may have a Suggested Fix. That means the system found a possible location, but it needs someone to verify it.

If there is no suggestion, manually fix the CSV by improving the address or entering Latitude and Longitude directly.

How to review a Suggested Fix

  1. Click 1. Check Map Location. Google Maps will open.
  2. Make sure the pin is in the correct place.
  3. Click 2. Review Suggested Fix. GitHub will open.
  4. If the location is correct, approve the Suggested Fix.
  5. If the location is wrong, reject the Suggested Fix and manually fix the CSV.

Approving a Suggested Fix

Approve a Suggested Fix only if the map pin looks correct.

  1. Click 2. Review Suggested Fix. GitHub will open.
  2. Click the green Merge pull request button.
  3. Click Confirm merge.
  4. Wait a few minutes for the map to rebuild.
  5. Refresh this dashboard. The row should disappear from the Review Queue.

Branch cleanup: Approved Suggested Fix branches usually disappear automatically if the repository setting Automatically delete head branches is turned on. If GitHub still shows a Delete branch button after approval, you can click it to clean up the temporary branch.

Rejecting a Suggested Fix

Reject a Suggested Fix if the map pin is wrong or you are not sure it is correct.

  1. Click 2. Review Suggested Fix. GitHub will open.
  2. Click Close pull request. Do not merge it.
  3. If GitHub shows a Delete branch button after closing, click it to clean up the temporary branch.
  4. Go back to the CSV and fix the row manually.
  5. Upload the corrected CSV again.

The row will stay in the Review Queue until the CSV is corrected or the row is removed.

What does Delete branch mean?

Some Suggested Fixes create temporary GitHub branches with names like geocode-suggestion/row-1821.

Deleting one of these temporary Suggested Fix branches only cleans up GitHub. It does not delete the map data, the CSV, the main branch, or the staging branch.

Never delete main or staging.

Most staff will not need to find branches manually. If needed, branches can be viewed in GitHub under Code → Branches.

If no Suggested Fix appears

Suggested Fixes are normally created automatically after the map rebuilds.

If a row has suggested coordinates but no Suggested Fix appears yet, go to:

GitHub Actions → Create Review Fixes → Run workflow

Then refresh the dashboard and try again.

What happens when I approve?

The suggested Latitude and Longitude are saved to the CSV. The map rebuilds automatically. The row disappears from the Review Queue after the next successful build.

What happens when I reject?

Nothing changes on the map. The Suggested Fix is closed. The row stays in the Review Queue until someone manually corrects the CSV or removes the row.

Manually fixing a row in the CSV

Use View CSV Row to see the exact row.

Use Copy Search Text to copy a raw CSV value like 1820,1819.

Then click Edit CSV File, press Ctrl+F, paste the search text, and find the row quickly.

Fix the address or enter Latitude and Longitude directly. Then upload the CSV again.

What not to do

  • Do not rename the CSV file.
  • Do not delete the Latitude or Longitude columns.
  • Do not change column headers unless a developer also updates the build script.
  • Do not approve a Suggested Fix unless the location on Google Maps looks correct.
  • Do not enter coordinates in any format other than decimal degrees, such as 36.1627, −86.7816.
  • Do not delete main or staging branches.
