📋 How to Update the Map

1 Download the current Master CSV using the button below.
2 Edit the CSV in Excel — add clinics, update addresses, or fix errors.
3 Upload the updated CSV to GitHub. GitHub automatically rebuilds the map.
4 Check the Review Queue below. If rows appear there, a location needs human review.
5 For each row in the Review Queue, click 1. Check Map Location to verify the pin, then click 2. Review Suggested Fix. Approve it if the location is correct, or reject it and fix the CSV if it is wrong.
Full Instructions & Help ▸

What this dashboard is

This page is read-only. It shows the current state of the Impact Map data. No changes can be made from this page — all map updates are applied through GitHub, which requires repository write access.

Uploading the CSV

Use the Upload Updated CSV button below. It opens GitHub's file upload page directly. You will need to be logged in with a GitHub account that has write access to this repository.

After uploading, GitHub Actions will run automatically and rebuild the map within a few minutes. Refresh this dashboard once the build finishes.

Do not rename the CSV file. The file must stay named Master_Clinic_ImpactMap.csv.

Green / All Clear status

If the status indicator is green and the Review Queue shows zero items, no action is needed. All clinic locations are already on the map.

Review Queue — what it means

When a row appears in the Review Queue, it means the system could not automatically fill in coordinates with enough confidence. A human needs to verify the suggestion.

  • Has Suggestion: The system found a likely location. Check the map pin, then approve or reject the Suggested Fix.
  • No Suggestion: The system could not find a location at all. Fix the CSV by adding a better street address or by entering the Latitude and Longitude directly.

Approving a Suggested Fix

  1. Click 1. Check Map Location on the review card to open Google Maps and verify the pin.
  2. Click 2. Review Suggested Fix to open the Suggested Fix on GitHub.
  3. If the location looks correct, click the green Merge pull request button, then Confirm merge.
  4. The map rebuilds automatically after approval.

If no Suggested Fix appears in the search results, go to GitHub Actions → Create Geocode Review PRs → Run workflow to create it manually. Suggested Fixes are normally created automatically after each build.

Rejecting a Suggested Fix

If the suggested location looks wrong:

  1. Click 2. Review Suggested Fix to open it on GitHub.
  2. Scroll to the bottom and click Close pull request (do not merge).
  3. Fix the CSV by adding a correct address or entering the Latitude and Longitude directly, then re-upload the CSV.

What happens when I approve?

The suggested coordinates are saved to the CSV and the map is rebuilt automatically. The row disappears from the Review Queue on the next build.

What happens when I reject?

Nothing changes on the map. The Suggested Fix is closed and the row stays in the Review Queue until you upload a corrected CSV with a better address or the correct coordinates.

What not to do

  • Do not rename the CSV file.
  • Do not delete the Latitude or Longitude columns.
  • Do not change column headers unless a developer also updates the build script.
  • Do not approve a Suggested Fix unless the location on Google Maps looks correct.
  • Do not enter coordinates in any format other than decimal degrees (e.g. 36.1627, −86.7816).
